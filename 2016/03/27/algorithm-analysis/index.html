Algorithm Analysis

This article summaries the basics about algorithm analysis.

An algorithm is a clearly specified set of simple instructions to be followed to solve a problem.

Mathematical Background

Definitions

  1. $T(N) = O\left(f(N) \right )$ if there are positive constants $c$ and $n_0$ such that $T(N) \leq cf(N)$ when $N \geq n_0$.

  2. $T(N) = \Omega\left(f(N) \right )$ if there are positive constants $c$ and $n_0$ such that $T(N) \geq cf(N)$ when $N \geq n_0$.

  3. $T(N) = \Theta\left(f(N) \right )$ if and only if $T(N) = O\left(f(N) \right )$ and $T(N) = \Omega\left(f(N) \right )$.

  4. $T(N) = o\left(f(N) \right )$ if, for all positive constant $c$, there exists an $n_0$ such that $T(N) < cf(N)$ when $N > n_0$. Less formally, $T(N) = o\left(f(N) \right )$ if $T(N) = O\left(f(N) \right )$ and $T(N) \neq \Theta\left(f(N) \right )$.

The above relationships can be shown in the following chart.

Rule 1.
If $T_1(N) = O\left(f(N) \right )$ and $T_2(N) = O\left(g(N) \right )$, then

  • $T_1(N) + T_2(N) = O\left(f(N) + g(N) \right )$.
  • $T_1(N) \times T_2(N) = O\left(f(N) \times g(N) \right )$.

Rule 2.
If $T(N)$ is a polynomial of degree $k$, then $T(N) = \Theta\left(N^k \right )$.

Rule 3.
$\log^k{N} = O(N)$ for any constant $k$. This tells us that logarithms grow very slowly.

Typical growth rates

Function Name
$c$ Constant
$\log{N}$ Logarithmic
$\log^2{N}$ Log-squared
$N$ Linear
$N\log{N}$
$N^2$ Quadratic
$N^3$ Cubic
